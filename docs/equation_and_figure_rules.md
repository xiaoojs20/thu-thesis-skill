# 公式、图表与引用规则

> 本文件同时整合两类信息：
> - 论文 PDF 中实际使用的表达模式
> - `thuthesis-v7.5.0` 中对应的 LaTeX 落地写法

## 1. 公式写作规则

## 1.1 公式引入的固定顺序

该论文中，公式几乎总是按下面顺序出现：

1. 先用一句话说明“为什么需要这个公式”
2. 给出单个公式或一组公式
3. 若是一组公式，逐条说明各式作用
4. 用 `式中：` 解释变量、函数、集合或上下标
5. 再补一句说明该公式实现了什么建模功能

典型模式如下：

```text
……如式（2.31）-（2.34）所示。
式（2.31）描述……，式（2.32）构建……，式（2.33）计算……，式（2.34）拟合……。
式中：A 表示……；B 表示……；C 为……。
基于式（2.31）-（2.34）构建的……，实现了……。
```

## 1.2 变量解释规则

PDF 中的变量解释有 4 个稳定特征：

1. 用 `式中：` 起头。
2. 按变量出现顺序解释，不跳跃。
3. 同一长句中用分号 `；` 分隔多个变量。
4. 解释内容优先写“变量物理含义”，再写“取值范围/角色/上限”。

实际模式示例可抽象为：

```text
式中：X 表示……；Y 表示……，包含……；Z 为……，其上限为……。
```

## 1.3 多公式组解释规则

对于连续编号公式，不是“放出来就结束”，而是必须逐条交代。

推荐顺序：

1. `式（n）描述状态转移`
2. `式（n+1）定义漂移项/目标函数/中间量`
3. `式（n+2）计算扩散项/风险项/约束项`
4. `式（n+3）给出概率密度或最终形式`

这类写法特别适合：

- 随机过程建模
- 分层优化模型
- 多约束块的规划模型
- 风险评估公式链

## 1.4 公式后的解释重点

该论文在公式后不会只解释符号，还会解释“公式意义”。常见落点有：

1. 描述随机性
2. 保留时序相关性
3. 量化季节波动
4. 建立平衡约束
5. 降低计算复杂度
6. 支撑后续海量场景计算

因此未来写作时，公式后至少回答一个问题：

`这个公式在论文主线里具体起什么作用？`

## 1.5 LaTeX 公式格式约定

在 `thuthesis-v7.5.0/data/chap03.tex` 和 `thuthesis-v7.5.0/thusetup.tex` 中，公式格式约定很明确：

1. 单行公式用 `equation` 或 `equation*`
2. 多行公式用 `align`
3. 多行公式尽量在 `=` 处对齐
4. 公式引用使用 `\eqref{...}`
5. 微分符号通过自定义命令 `\dif`
6. 数学字体统一由 `math-font = xits` 控制
7. 单位推荐用 `siunitx`

推荐模板：

```latex
\begin{equation}
  ...
  \label{eq:model}
\end{equation}

\begin{align}
  a &= b + c \\
    &= d + e
\end{align}
```

## 2. 符号与变量说明规则

## 2.1 章节级符号表

成品论文第 3 到第 5 章都单独设置了 `本章主要符号表`，说明当章节模型复杂时，章节前置符号表比把全部变量压在首次公式后更清晰。

适用条件：

- 本章变量数量多
- 有重复使用的上标/下标/集合
- 同时存在规划变量、状态变量、参数和风险指标

## 2.2 全文符号说明区

LaTeX 项目使用 `denotation` 环境：

```latex
\begin{denotation}[3cm]
  \item[PI] 聚酰亚胺
  \item[$E_a$] 化学反应的活化能（Activation Energy）
\end{denotation}
```

可总结出 3 条规则：

1. 缩略语和变量都可以进入符号表。
2. 中文释义在前，英文全称放括号中。
3. 数学变量允许直接写在 `\item[...]` 中。

## 2.3 变量说明的推荐分层

未来写作时，变量说明建议分三层：

1. 全文符号表：高频缩略语、常用符号
2. 章节符号表：本章核心变量和集合
3. 公式后 `式中：`：只解释当前公式新增或易混淆符号

## 3. 插图规则

## 3.1 图的功能分层

该论文图的用途非常稳定：

1. 引言章：背景认知图、研究框架图、问题定义图
2. 方法章：流程图、拓扑图、统计曲线、建模示意图
3. 实验章：系统结构图、运行结果图、平衡曲线、灵敏度图

换句话说：

- 引言图回答“为什么研究”
- 方法图回答“怎么建模”
- 结果图回答“方法效果如何”

## 3.2 图题写法

图题几乎都是“名词短语 + 对象限定”，而不是一句完整结论。

实际图题模式：

1. `某系统拓扑结构`
2. `某方法框架`
3. `某结果对比分析`
4. `某月平衡结果`
5. `某灵敏度分析`

不要把结论直接塞进图题。结论应写在正文解释里。

## 3.3 图前图后的正文动作

图表附近的正文动作也很固定：

1. 图前一句说明“展示什么”
2. 图后一句或几句说明“从图中看出什么”
3. 若有对比，再补“与方案 A 相比”

推荐模板：

```text
图 x 展示了……。
由图可知，……。
相较于……，……。
```

## 3.4 LaTeX 插图格式

`thuthesis-v7.5.0/data/chap02.tex` 给出了标准写法：

```latex
\begin{figure}
  \centering
  \includegraphics[width=0.5\linewidth]{example-image-a.pdf}
  \caption{示例图片标题}
  \label{fig:example}
\end{figure}
```

图环境的硬规则：

1. 用 `figure` 环境
2. 先 `\centering`
3. 再 `\includegraphics`
4. `\caption` 在前，`\label` 在后
5. 图文件默认从 `figures/` 目录搜索，因为 `\graphicspath{{figures/}}`

## 3.5 多子图格式

模板推荐 `subcaption`：

```latex
\subcaptionbox{分图 A\label{fig:subfig-a}}
  {\includegraphics[width=0.35\linewidth]{...}}
```

适用场景：

- 原始数据 vs 模拟数据
- 不同方案并列
- 不同季节/区域对比

## 4. 表格规则

## 4.1 表的功能

表格在这篇论文里主要承担 4 件事：

1. 参数设置
2. 方案描述
3. 优化结果对比
4. 文献综述归纳

如果信息是“结构化数值对比”，优先用表，不要强行画图。

## 4.2 表题写法

表题与图题一样，采用名词短语：

- `对比方案描述`
- `关键经济技术参数设置`
- `不同方案的优化结果对比分析`

## 4.3 LaTeX 表格格式

模板明确推荐三线表：

```latex
\begin{table}
  \centering
  \caption{三线表示例}
  \begin{tabular}{ll}
    \toprule
    ...
    \midrule
    ...
    \bottomrule
  \end{tabular}
  \label{tab:three-line}
\end{table}
```

规则如下：

1. 优先三线表，不使用竖线网格表
2. `\caption` 放表格上方
3. `\label` 放在 `\caption` 之后
4. 有附注时用 `threeparttable`
5. 跨页表用 `longtable`

## 5. 引用与参考文献规则

## 5.1 成品论文的引用风格

成品论文正文采用数字顺序编码制：

- 中文叙述中直接接 `[1]`、`[2]`
- 连续引用写成 `[9-10]`
- 引用与正文紧密绑定，不单独成句

## 5.2 LaTeX 项目的实现方式

`thuthesis-v7.5.0/thusetup.tex` 中默认配置为：

```latex
\usepackage[sort]{natbib}
\bibliographystyle{thuthesis-numeric}
```

主文档中：

```latex
\bibliography{ref/refs}
```

可直接总结为：

1. 默认 `natbib + BibTeX`
2. 默认样式 `thuthesis-numeric`
3. 文献库放在 `ref/refs.bib`
4. 常用命令是 `\cite`、`\citet`、`\citep`

## 5.3 可复用的落地规范

未来写作时，建议统一遵守：

1. 全文只选一种引用风格，不要混用。
2. 方法或结论句后再引用，不要让引用代替论证。
3. 图表、公式、文献都使用统一标签前缀：
   - 图：`fig:`
   - 表：`tab:`
   - 公式：`eq:`
4. 文本解释中优先写：
   - `如图 3.2 所示`
   - `由表 4.1 可知`
   - `如式（2.18）所示`
