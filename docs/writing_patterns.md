# 论文写作模式分析

> 目标：把该论文的写法抽象成未来可复用的写作规则，而不是停留在“写得比较正式”这种空话。

## 1. 段落推进模式

该论文最常见的段落推进顺序是：

1. 先给背景事实或工程现象
2. 再指出矛盾、缺口或困难
3. 然后引出“本文/本章针对什么”
4. 最后说明方法、作用或结果

高频连接词具有明显规律：

| 连接词/短语 | 全文出现次数 |
| --- | ---: |
| `针对` | 142 |
| `首先` | 28 |
| `其次` | 7 |
| `最后` | 12 |
| `基于此` | 19 |
| `由图` | 25 |
| `由表` | 9 |
| `综上所述` | 3 |

这说明它采用的是非常典型的工科论文推进方式：

- 用 `针对...` 开启问题导向
- 用 `首先/其次/最后` 组织方法链条
- 用 `基于此` 连接“前文方法”和“后文应用”
- 用 `由图/由表` 驱动结果解释

### 1.1 常见句式模板

最常见的句式可以直接归纳为下面 6 类：

1. `针对 X 问题，本文提出 Y 方法。`
2. `首先，……；其次，……；最后，……。`
3. `基于此，构建/建立/实现……。`
4. `如图 x 所示，……。`
5. `由表 x 可知，……。`
6. `通过上述对比/分析，证明/说明……。`

未来写作时，正文段落尽量保持“一段只完成一个动作”，不要在同一段里混写背景、综述、方法和结果。

## 2. 引言写法

## 2.1 引言的内部顺序

这篇论文的引言不是“背景 -> 综述 -> 贡献”三步，而是更细的八步：

1. 国家战略/行业背景
2. 研究对象的现实矛盾
3. 潜在解决资源或关键技术
4. 更大的应用场景或系统形态
5. 核心科学问题
6. 子问题拆解与技术难点
7. 研究意义
8. 文献综述、研究思路、主要工作

其中 `1.1 论文研究概述` 内部又是：

- `研究背景`：先铺现实场景，再铺技术资源，再上升到系统级对象
- `关键问题`：把总问题拆为 3 个子问题，再给每个子问题对应的技术难点
- `研究意义`：按 3 条左右的“理论/方法/应用意义”展开

## 2.2 引言的具体写法规则

### 规则 1：背景要从“大环境”逐步压缩到“本文问题”

该论文的背景写法是：

1. 碳中和、新型电力系统等宏观背景
2. 可再生能源并网后的季节性失衡问题
3. 季节性储能/综合能源单元等应对资源
4. 回到论文核心问题

不要一开头就直接写模型。

### 规则 2：关键问题必须拆成子问题

该论文不是只写一句“存在若干挑战”，而是明确写成：

1. 子问题 1
2. 子问题 2
3. 子问题 3

并且每个子问题下再给 2 个技术难点。

这是非常强的博士论文写法，因为它把后续章节目录提前合理化了。

### 规则 3：综述要按主题，不按作者流水账

文献综述部分不是“某某研究了……某某又研究了……”，而是按研究主题分块：

- 可再生能源季节特性建模
- 季节性储能建模
- 考虑储能的优化规划
- 风险评估方法

每个主题内部再总结现状、方法类型、已有不足。

### 规则 4：引言结尾要直接对齐全文结构

`1.3.2 主要工作` 明确写出：

- 第 1 章做什么
- 第 2 章做什么
- 第 3 章做什么
- 第 4 章做什么
- 第 5 章做什么
- 第 6 章做什么

这一步不是摘要式复述，而是为了给整篇论文建立导航。

## 3. 摘要写法

## 3.1 中文摘要模式

中文摘要非常稳定地分成 6 个写作动作：

1. 先写研究背景和总矛盾
2. 再写本文聚焦的核心科学问题
3. 然后按工作 1、2、3、4 依次概括
4. 每个工作都写“针对什么 -> 提出什么 -> 实现什么”
5. 最后给总括性总结
6. 结尾给 5 个关键词

### 摘要句法模板

可直接复用为：

1. `……为……带来巨大挑战。`
2. `针对“……”这一科学问题，本文……。`
3. `针对 A，本文……。`
4. `针对 B，本文……。`
5. `针对 C，本文……。`
6. `综上所述，本文的工作为……提供了……。`

## 3.2 摘要的硬规则

该论文摘要体现出以下可直接复用的硬规则：

1. 不写“第 1 章……第 2 章……”。
2. 不写文献综述。
3. 不展开公式、参数、图表编号。
4. 不写泛泛的“具有重要意义”，而是写方法产出和应用价值。
5. 每个贡献压缩到 2 到 4 句。

## 4. 结论写法

## 4.1 结论结构

结论采用“三段式”：

1. 开头一段：回收研究背景、总问题和论文整体技术路线
2. 中间主体：按 4 项成果编号总结
3. 结尾两段：总体发现 + 应用价值/展望

结论不是简单复制“本章小结”，而是重新站在全文层面组织。

## 4.2 结论的编号方式

编号直接对应核心章节成果：

1. 提出季节不确定性分析与建模方法
2. 提出全景时序运行模拟方法
3. 提出考虑季节性储能优化配置的规划方法
4. 提出季节性不平衡风险评估方法

这类编号有两个优点：

1. 和正文主线完全对齐
2. 每一条都能写成“问题 -> 方法 -> 结果 -> 价值”

## 4.3 结论的推荐句法

每个编号段建议固定四步：

1. 说明研究对象或挑战
2. 说明本文提出的方法
3. 说明方法做到了什么
4. 说明验证了什么或有何价值

示意模板：

```text
（n）提出了……
针对……问题，本文……。
基于……，建立/构建……。
最后通过……算例验证了……。
```

## 5. 本章小结写法

第 2 章到第 5 章的 `本章小结` 都遵循同一模式：

1. 先重述本章面对的挑战
2. 再按 `首先/其次/基于此/最后` 回顾方法链
3. 最后说明算例验证结果

这说明章末总结不能只写“本章完成了什么”，而要写清：

- 为什么需要这一章
- 这一章的方法链条是什么
- 这一章的方法是否被验证

## 6. 可直接复用的写作模板

### 6.1 背景段模板

```text
随着……，系统面临……挑战。
一方面，……；另一方面，……。
因此，如何……成为……领域的重要问题。
```

### 6.2 方法引入段模板

```text
针对……问题，本文提出……方法。
首先，……；
其次，……；
基于此，……；
最后，通过……验证……。
```

### 6.3 结果解释段模板

```text
如图 x 所示，……。
由图可知，……。
相较于方案 A，方案 B 在……方面……。
通过上述对比分析，证明/说明……。
```
