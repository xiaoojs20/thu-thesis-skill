# 方法章节模板

用于模型、方法、仿真或优化规划类章节。

## 推荐结构

这类章节有一条较强的骨架约束：

1. `概述`
2. `主要内容`
3. `算例分析`
4. `本章小结`

在实际写作中，`主要内容` 通常会展开为 `x.2-x.4` 这样的多个连续小节。

```text
第x章 [方法/模型标题]
x.1 概述
x.2 定义、符号或问题分析（主要内容）
x.3 核心模型或方法（主要内容）
x.4 扩展框架、约束或优化模型（主要内容）
x.5 算例分析
x.6 本章小结
```

## 写作说明

### x.1 概述

篇幅强约束：`概述` 至少应约占编译后 PDF 的 2/3 页。不要写成单段短说明；若尚未编译，应写成多个完整段落，覆盖问题承接、章节目标、方法路线、主要内容安排和验证思路。

建议交代：

1. 本章目标
2. 现有方法存在的不足
3. 本章将构建什么

模板：

```text
针对……问题，现有方法通常难以……。
因此，本章提出……方法/模型。
本章的核心内容包括：……。
```

### x.2 定义、符号或问题分析

这一节属于章节的 `主要内容` 区块，主要用于在公式密集出现前稳定符号空间。

可包含：

- 关键定义
- 符号表或符号说明
- 如需，可包含问题分解

### x.3 核心模型或方法

这一节同样属于章节的 `主要内容` 区块。

每个小节：

1. 说明为何需要该模型块
2. 给出公式
3. 解释公式角色
4. 补充 `式中：`
5. 说明该块达成什么

模板：

```text
为描述……，构建如式（x.1）-（x.3）所示的模型。
式（x.1）描述……，式（x.2）定义……，式（x.3）计算……。
式中：……。
基于上述模型，可以实现……。
```

### x.4 扩展框架、约束或优化模型

这一节是 `主要内容` 区块的最后一部分，用来把前面内容总装成完整框架：

- 目标函数
- 运行约束
- 平衡约束
- 框架集成

若该节为优化规划小节，如 `考虑……协同的……优化规划模型`，请打开 `optimization-model-pattern.md` 并按该更细粒度的模式写作：

1. 承继前文子模型的开篇段
2. 规划边界复杂时给出一幅耦合/框架图
3. `x.x.1 目标函数`，用 `（1）（2）（3）` 拆分目标块
4. `x.x.2`、`x.x.3` 等主要约束类别
5. 每个主要约束类别内部，用 `（1）（2）（3）` 拆分物理模块
6. 收尾段说明完整模型及求解器类型

### x.5 算例分析

打开 `experiment.md` 并按该模板执行。

### x.6 本章小结

篇幅强约束：`本章小结` 至少应约占编译后 PDF 的 2/3 页。不要只用 3 到 5 句话收束本章；应分段回扣本章问题、归纳方法链条、说明算例验证结论，并交代其对全文主线或后续章节的支撑。

模板：

```text
首先，本章针对……提出了……。
其次，构建了……。
基于此，建立了……。
最后，通过算例分析验证了……。
```

## 最终检查

- 本章明确遵循 `概述 -> 主要内容 -> 算例分析 -> 本章小结`
- `概述` 与 `本章小结` 各自至少约占编译后 PDF 的 2/3 页；若无法编译，则各自展开为多个完整段落
- 每个模型块解决一个明确的子问题
- 公式在出现前有引入
- 符号受控
- 算例分析验证的是方法，而非仅展示数字
