# 论文结构

## 全文结构

本文件只描述可复用的结构规律。诸如 `规划`、`风险`、`运行`、`建模`、`实验`、`案例分析` 等章节名称只是结构角色示例，不是必须使用的主题词。

全文应保持单一主问题研究线：

1. 前置材料
2. 中文摘要
3. 英文摘要
4. 目录、插图清单、附表清单
5. 符号/缩略语说明
6. 第 1 章 引言
7. 第 2 章 方法基础 A
8. 第 3 章 方法基础 B 或求解框架
9. 第 4 章 核心规划/模型章
10. 第 5 章 风险/扩展/评估章
11. 第 6 章 结论
12. 参考文献与后置材料

这不是"多篇论文拼接式"结构。每一章都应解决同一主问题链条中的一个环节。

在确定最后一章编号前，先判断论文层次：

- 硕士论文：论文通常以第 5 章结尾，因此结论常为第 5 章
- 博士论文：论文通常以第 6 章结尾，因此结论常为第 6 章
- 若学位层次未知，将结论章编号保留为占位符，不要写死

不要照抄样本论文的章节主题，应改写为用户自己的研究对象，例如：

- `基础理论`
- `方法设计`
- `系统实现`
- `实验验证`
- `扩展分析`

可将正文主线概括为：

1. 第 1 章回答"为什么研究、前人做到哪里、本文准备怎么做"
2. 第 2 章回答"研究对象或基础问题如何刻画"
3. 第 3 章回答"求解、模拟、识别或实现方法如何建立"
4. 第 4 章回答"主体模型、核心策略或系统设计如何形成"
5. 第 5 章回答"扩展分析、风险控制、泛化验证或强化约束如何完成"
6. 结论章回答"全文贡献和验证结果是什么"

## 章节骨架

### 第 1 章：引言

采用：

1. `1.1 研究概述`
2. `1.2 国内外研究现状`
3. `1.3 研究思路与主要工作`

`1.1` 内部保持：

1. 研究背景
2. 关键问题
3. 研究意义

`1.3` 内部保持：

1. 研究思路
2. 逐章主要工作

### 技术章节

对结论章之前的技术章节，强制保持以下模式：

1. `x.1 概述`
2. `x.2-x.4 主要内容`
5. `x.5 算例分析`
6. `x.6 本章小结`

这是强约束，不是松散建议。

含义如下：

- `概述`：章节目标、问题与内容地图
- `主要内容`：一个或多个连续小节，承载定义、模型、方法、框架、约束、算法或推导
- `算例分析`：通过案例、实验、仿真或对比研究进行验证
- `本章小结`：章级回扣问题、方法链条与验证结果

`主要内容` 不必以字面节标题出现，但必须占据 `概述` 与 `算例分析` 之间的中间区块。

`概述` 与 `本章小结` 的篇幅都是强约束：每一节至少应约占编译后 PDF 的 2/3 页。起草时不得把 `概述` 写成一段背景说明，也不得把 `本章小结` 写成一段短总结；若尚未编译 PDF，应按多个完整段落展开，使其在编译后大致达到该版面量。

常见的 `x.2-x.4 主要内容` 组织方式有三类：

1. `定义 / 分类 / 指标 / 挑战`
2. `模型 / 目标函数 / 约束 / 求解框架`
3. `模块分析 / 分量建模 / 子系统设计 / 扩展机制`

若某一节承担"模型建立"功能，三级标题优先按以下顺序排布：

1. 目标或问题定义
2. 变量与参数
3. 目标函数或核心机理
4. 核心约束或关键模块
5. 框架、流程或求解方式

### 论文层面的结论章

对论文层面的结论章，强制保持以下模式：

1. 仅设章标题：`第x章 结论`
2. 开篇总述段
3. 编号主要贡献
4. 收尾段

这同样是强约束。

若为硕士论文，`x` 常为 `5`。

若为博士论文，`x` 常为 `6`。

结论章内部不要设 `x.1`、`x.2`、`x.3` 或任何小节标题。

## 节内组织顺序

当某一节引入模型时，优先按以下顺序：

1. 问题或目标定义
2. 变量或符号澄清
3. 目标函数或主要机理
4. 约束或子模块
5. 建模意义

当某一节引入实验/算例时，优先按以下顺序：

1. 算例设置
2. 方案定义
3. 参数或边界条件
4. 结果对比
5. 解读
6. 结论

## 常见小节类型

典型小节模式：

1. `定义 / 指标 / 挑战`
2. `模型 / 目标函数 / 约束`
3. `边界条件说明 / 结果对比分析 / 灵敏度分析`

## 章节篇幅建议

根据典型工科博士论文的统计，推荐比例如下：

| 章节 | PDF 页数 | 中文字符数估计 | 正文占比 |
| --- | ---: | ---: | ---: |
| 第 1 章 引言 | ~28 | ~22,000 | ~30% |
| 第 2 章 方法 1 | ~26 | ~15,000 | ~19% |
| 第 3 章 方法 2 | ~23 | ~12,000 | ~16% |
| 第 4 章 核心方法/模型 | ~25 | ~12,000 | ~15% |
| 第 5 章 扩展分析 | ~24 | ~12,000 | ~16% |
| 第 6 章 结论 | ~4 | ~3,000 | ~4% |

可直接提炼的写作配比：

- 引言是最长章，约占正文 30%。
- 每个核心技术章占 15% 到 20%。
- 结论明显短于技术章，控制在正文 4% 到 6%。

硕士论文的第 2 章到第 4 章还应执行更具体的编译后篇幅控制：

- 每一章编译后大约控制在 25 页，最多不超过 30 页。
- 每一章中的 `算例分析 + 本章小结` 合计控制在 10-15 页。
- 这是较强约束；若由于图表、公式、学校模板或研究内容差异导致无法满足，应在编译后明确记录实际页数、偏离量和原因。
- 每次完成 LaTeX 编译后，都要检查上述页数，而不是只在定稿前检查一次。

技术章节中，`算例分析 + 本章小结` 通常承担收束和验证职责，应作为独立的篇幅预算单独控制。若研究内容确实需要更多结果展示，应优先：

1. 合并相近验证目标
2. 将局部细节下沉为段落说明或 `（1）（2）（3）`
3. 避免把大量边缘分析直接扩成新的三级小节

## 图表密度建议

按插图清单和附表清单统计的典型分布：

| 章节 | 图数 | 表数 | 图表特点 |
| --- | ---: | ---: | --- |
| 第 1 章 | ~7 | ~4 | 背景图、框架图、综述对比表 |
| 第 2 章 | ~20 | ~8 | 方法流程图、统计图、建模参数表最多 |
| 第 3 章 | ~14 | ~8 | 框架图、拓扑图、方案对比图表 |
| 第 4 章 | ~17 | ~4 | 结构图、过程图、方案结果图 |
| 第 5 章 | ~13 | ~3 | 风险图、灵敏度图、方案设置表 |

这说明：

1. 引言图表只负责"建立问题"和"展示框架"，不负责堆结果。
2. 方法章图表最密，用于解释流程、变量关系、统计规律。
3. 实验章和规划章图多于表，强调结果可视化。
4. 表格主要承担"参数、方案、结果对比"三种功能。

概括建议：

- 引言：图和表较少；仅用于背景和框架。
- 方法章：图密度最高；用于流程、拓扑和统计规律。
- 规划与实验章：结果图多于表。
- 表主要承载：
  - 方案描述
  - 参数
  - 结果对比

## 可复用提纲骨架

```text
第1章 引言
  1.1 研究概述
    1.1.1 研究背景
    1.1.2 关键问题
    1.1.3 研究意义
  1.2 国内外研究现状
  1.3 研究思路与主要工作

第2章 研究对象分析与基础建模
  2.1 概述
  2.2 对象定义与指标
  2.3 建模方法
  2.4 分模块建模
  2.5 算例分析
  2.6 本章小结

第3章 求解框架或运行模拟方法
第4章 核心模型与优化规划
第5章 风险评估或扩展分析
第x章 结论
```

套用该骨架时，要保留推进顺序和节级逻辑，但章节标题必须完全围绕目标论文主题重写。

对技术章节，不可打破的顺序是：

1. `概述`
2. `主要内容`
3. `算例分析`
4. `本章小结`

对论文层面的结论章，不可打破的规则是：

1. 不设节标题
2. 不设小节标题
3. 直接在章标题下以连续段落加编号贡献项展开
