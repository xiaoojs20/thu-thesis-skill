# 算例与案例研究模式

## 默认实验逻辑

按以下顺序展开：

1. 说明验证目标
2. 描述案例系统或数据集
3. 定义对比方案
4. 给出参数或边界条件
5. 呈现结果
6. 解读结果背后的机理
7. 说明对比证明了什么

这不是简单的"实验设置 + 结果"两段式，而是完整的验证链。

## 常见结构

### 模式 A：方法验证

用于新模型或分解方法：

1. 实验目标
2. 数据来源与计算环境
3. 逐模块结果
4. 整体重构或最终对比
5. 结论

这类结构适合先验证中间模块，再验证整体重构效果。

### 模式 B：方案对比

用于优化、仿真或求解框架：

1. 方案设置
2. 测试系统说明
3. 参数设置
4. 结果对比
5. 解读
6. 结论

这类结构适合先把方案定义清楚，再分别做机制验证和规模验证。

### 模式 C：风险或鲁棒性分析

用于不确定性、CVaR 或鲁棒规划：

1. 方案设置
2. 场景或不确定性配置
3. 基础结果
4. 灵敏度分析
5. 大规模系统验证
6. 结论

这类结构适合先验证风险项是否起作用，再分析参数变化规律。

## 可复用节骨架

强约束：每个 `算例分析` 节下面最多设置五个直接三级小节。若 `算例分析` 编号为 `x.y`，应将内容合理安排在 `x.y.1` 至 `x.y.5` 之内；例如 `2.5 算例分析` 最多安排到 `2.5.5`，`3.4 算例分析` 最多安排到 `3.4.5`。更多内容应按验证目标合并到现有小节，或使用段落、图表说明、`（1）（2）（3）` 层级承载。

```text
x.5 算例分析
x.5.1 算例设置
  （1）测试系统说明
  （2）方案定义
  （3）参数设置
x.5.2 基础结果对比
x.5.3 运行过程或平衡特性分析
x.5.4 灵敏度分析（可选）
x.5.5 大规模系统验证（可选）
```

## 典型展开方式

若是新建模方法章节，优先采用：

1. 分模块结果
2. 中间量验证
3. 整体重构或综合对比

若是新求解框架或新规划模型章节，优先采用：

1. 先给方案设置
2. 先用小系统验证机制正确性
3. 再用实际或大系统验证可扩展性

若是风险、鲁棒性或机会约束章节，优先采用：

1. 先给算例设置表
2. 再比较基础结果
3. 再做灵敏度分析
4. 最后在更大系统上验证有效性

## 算例开头模板

算例节开头至少回答四个问题：

1. 用什么系统
2. 为什么选这个系统
3. 比较哪些方案
4. 在什么环境下计算

推荐模板：

```text
本节基于……系统进行算例分析，以验证……。
共考虑……组方案，对比设置如表 x 所示。
所有算例通过……实现，并在……环境下测试。
```

## 结果段模板

```text
表 x 对比了方案 A 和方案 B 的结果。
由表可知，方案 B 在……方面优于方案 A，主要体现在……。
相应地，……由……变化为……。
通过上述对比分析，证明了……。
```

## 更具体的结果分析节奏

对于优化、规划或可靠性导向的实验，若材料适合，优先采用以下顺序：

1. 介绍结果表或图，并重述验证目的
2. 若可行性、可靠性或充裕度是核心论点，则优先分析
3. 若相关，其次分析经济性、效率或成本
4. 给出具体的增减数值
5. 以对比证明了什么结尾

可用措辞：

```text
表 x 基于……对比分析了不同方案的优化结果，以验证……。
由对比结果可知，……
在……方面，……
在……方面，……
通过上述对比分析，证明了……。
```

## 图解读模板

```text
图 x 展示了……。
由图可知，……。
系统在……阶段表现出……规律。
这说明……。
```

图像结果解释不应重复图题，而应解释变化规律，例如：

1. 峰谷出现于什么阶段、季节或工况
2. 充能和放能分别发生在何处或何时
3. 方案差异来自什么机制
4. 结果是否符合预期规律

## 图/表排序模式

案例研究小节内部的首选顺序：

1. 方案表
2. 拓扑图
3. 参数表
4. 结果表
5. 动态结果图

这是最稳妥的工科算例组织方式。

## 什么才算扎实的实验节

实验节应回答：

1. 验证什么
2. 与什么基线对比
3. 在什么条件下
4. 发生了什么变化
5. 为何变化
6. 这对论文论点意味着什么
