# 优化-规划模型总装模式

当论文包含如下小节时使用：

- `x.x 考虑……协同的……优化规划模型`
- `x.x 考虑……的……优化配置模型`
- `x.x ……优化模型`
- `x.x ……规划模型框架`

本模式用于将前面的定义、子模型或机理分析总装为一个完整的优化/规划问题。

## 默认小节节奏

优先采用以下顺序：

1. 开篇总装段
2. 建模边界复杂时给出一幅框架/耦合图
3. `x.x.1 目标函数`
4. `x.x.2` / `x.x.3` / `x.x.4` 等主要约束类别
5. 每个主要块内部使用加粗的中文圆括号子项，如 `\textbf{（1）投资成本}`
6. 收尾句或过渡到实验节

## 开篇总装段

`x.x` 的开头段通常承担两项工作：

1. 承继前文子模型
2. 宣告完整的优化/规划模型已建立

模板：

```text
基于上述构建的……模型，本文进一步建立了考虑……协同的……优化规划模型。
该模型综合考虑……、……与……之间的耦合关系，以实现……的协同优化配置。
```

## 先给框架图

若规划边界包含多个子系统、能源形式、传输链路或储能路径，优先在开篇段之后立即放置一幅框架或耦合图。

其作用是澄清：

- 系统边界
- 资源输入
- 转换设备
- 储能单元
- 网络或传输链路
- 所供负荷

## 目标函数写作

`目标函数` 下优先采用：

1. 一个顶层总目标
2. 中文圆括号分解，如 `（1）投资成本`、`（2）运行成本`
3. 每个块的物理含义解释
4. 必要时解释重要变量、上标或下标

重点不仅是列出公式，还要让优化方向清晰可读。

强制性格式：

1. 每个子项使用中文圆括号编号，如 `（1）`、`（2）`、`（3）`
2. 子项标题加粗
3. 加粗子项标题独占一行或一段
4. 解释从下一段开始，不要接在同一行
5. 第一段解释保持首行缩进
6. 若同一子项下有多段解释，每段均保持首行缩进，不要顶格

推荐模板：

```text
\textbf{（1）投资成本}

　　如式（x.2）所示，……。
　　式（x.2）-（x.4）分别表示……。

\textbf{（2）运行成本}

　　如式（x.5）所示，……。
```

## 约束层次

这是推荐模式，不是硬规则。

当模型较大或跨越多个物理子系统时，优先采用两层约束组织：

1. 小节级别的大类，如 `其他能源系统运行约束`、`电力系统运行约束`
2. 每个大类内部的物理模块分解，如 `（1）火电机组……约束`、`（2）可再生能源……约束`

仅当该结构能显著提升可读性时使用。不要强行套用到更适合扁平布局的小模型上。

每个物理模块分解项保持同样的强制性格式：

- 加粗中文圆括号 lead
- 下一段解释
- 每段解释保持首行缩进

## 公式组解释

当多条相邻公式共同描述一个块时，优先将它们作为公式组解释，而非逐条孤立叙述。

典型模式：

```text
式（x.1）-（x.3）描述……，其中式（x.1）定义……，式（x.2）表示……，式（x.3）计算……。
```

或

```text
式（x.4）和式（x.5）构建了……约束。
```

以下情况使用该模式：

- 公式属于同一机理块
- 作为集合更容易理解
- 分开叙述会显得重复

以下情况不要强行套用：某条公式在概念上独立，值得单独叙述。
