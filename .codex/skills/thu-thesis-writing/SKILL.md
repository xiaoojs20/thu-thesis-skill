# 清华风格工科论文写作

当用户希望论文文本遵循从高质量清华风格工科博士论文中提炼出的结构与语气时，使用本 skill。

本 skill 明确以"写作形式优先"，而不是"样本主题优先"：

- 学习章节组织、段落逻辑、公式叙述、图表规范和实验章节骨架
- 不要继承样本论文的研究主题、领域假设、系统名称、数据集、变量或结论
- 不要把电力系统、能源系统、规划、储能、风险等样本主题词强行带入其他论文

## 本 Skill 强制保持的写法

1. 问题驱动的论文结构，而非文章拼接式结构。
2. 强技术章节骨架：`概述 -> 主要内容 -> 算例分析 -> 本章小结`。
3. 正式的中文工科论文学术散文。
4. 显式公式解释风格，含 `式中：`。
5. 清晰的图与表分工。
6. 可复用的实验节模式。
7. 扎实的引言与结论支架。
8. 章级审阅检查，确保结构完整与局部逻辑通顺。
9. 主题隔离：提取写作形式，而非研究主题。

## 适用场景

本 skill 适用于：

- 论文章节起草
- 章节提纲拟定
- 论文章节改写
- 论文章节审阅
- 论文节段审阅
- 摘要写作
- 引言写作
- 方法论/模型章写作
- 实验/案例研究节写作
- 结论写作
- 将粗略笔记转换为论文学术散文

如果目标论文包含优化/规划模型小节，应将 `optimization-model-pattern.md` 视为默认写法，而不是可选参考。

不适用于：

- 目标风格迥异的期刊论文
- 非正式报告
- 幻灯片
- 代码或实现文档

## 使用流程

1. 先识别目标产出类型：
   - 摘要
   - 引言
   - 方法论/模型章
   - 实验节
   - 结论
   - 完整章节提纲
   - 章级审阅
   - 节段审阅
2. 先判断论文层次：
   - 硕士论文：结论章通常是第 5 章
   - 博士论文：结论章通常是第 6 章
   - 如果学位层次未知，不要提前写死结论章编号
3. 只读取与当前任务相关的文件：
   - 总体结构 -> `thesis-structure.md`
   - 行文逻辑与语气 -> `writing-patterns.md`
   - 公式、图、表 -> `equation-rules.md`
   - 算例/实验章节 -> `experiment-pattern.md`
   - 优化规划模型小节 -> `optimization-model-pattern.md`
   - LaTeX 编译与清理命令 -> `latex-build-commands.md`
   - 最终自查 -> `checklist.md`
   - 现成模板骨架 -> `chapter-templates/*.md`
4. 如果是起草任务，先列提纲，再写正文。
5. 对技术章节，强制保持以下四段骨架顺序：
   - `概述`
   - `主要内容`
   - `算例分析`
   - `本章小结`
6. 将 `主要内容` 视为章节主体，可展开为定义、模型、约束、框架、算法等多个连续小节，但必须位于 `概述` 与 `算例分析` 之间。
7. 对任何 `算例分析` 节，其下属三级小节数量是强约束：若 `算例分析` 编号为 `x.y`，应合理组织在 `x.y.1` 至 `x.y.5` 之内。例如 `2.5 算例分析` 最多安排到 `2.5.5`，`3.4 算例分析` 最多安排到 `3.4.5`。内容较多时，应按验证目标合并相近分析、下沉为段落或使用 `（1）（2）（3）` 层级承载，使算例结构保持在五个直接三级小节以内。
8. 对每个技术章节，`概述` 与 `本章小结` 的篇幅都是强约束：各自至少应约占编译后 PDF 的 2/3 页；如果无法实际编译估算，不得写成单段短文本，应按多个完整段落展开到接近该版面量。
9. 对硕士论文，第 2 章到第 4 章的技术章篇幅是较强约束：每章编译后大约控制在 25 页，最多不超过 30 页；其中 `算例分析 + 本章小结` 合计控制在 10-15 页。若客观上无法满足，应明确说明原因和偏离量。
10. 每次完成 LaTeX 编译后，都要检查第 2-4 章总页数及其 `算例分析 + 本章小结` 页数是否落在上述范围内。
11. 全文默认按"问题 -> 方法 -> 结果"的顺序起草。
12. 如果论文中出现 `考虑……协同的……优化规划模型`、`……优化配置模型` 等"模型总装段"，先读 `optimization-model-pattern.md`，并默认采用该写法。
13. 对论文层面的 `结论` 章，不要再设节或小节；无论结论是第 5 章还是第 6 章，都应写成"章节标题 + 连续段落 + 编号贡献项"。
14. 公式默认按"先说明用途 -> 再给公式 -> 再给 `式中：` -> 再说明建模含义"的顺序解释。
15. 图表标题默认使用名词性表述，核心结论放在正文中说明。
16. LaTeX 工程中的图优先使用 PDF；如果图不是 PDF，要明确提醒用户。
17. 每章技术章节结尾都要有 `本章小结`，并重述：
    - 本章面临的挑战
    - 本章构建的方法链条
    - 算例验证了什么
18. 如果是审阅任务，使用 `checklist.md` 作为检查表，并明确指出：
    - 缺少章引言
    - 缺少章总结
    - 章概述或章总结短于编译后 PDF 约 2/3 页
    - 硕士论文第 2-4 章编译后页数超出约 25 页或超过 30 页
    - `算例分析 + 本章小结` 编译后超出 10-15 页
    - `算例分析` 直接下属小节未合理组织在 `x.y.1` 至 `x.y.5` 之内
    - 公式后缺少变量解释
    - 图/表引用缺失或薄弱
    - 实验结构不完整
    - 节间过渡薄弱
19. 输出前务必再跑一遍 checklist。

## 主题隔离规则

本 skill 背后的来源材料来自某一篇具体样本论文，但该样本只用于提炼写作形式。

使用本 skill 时：

1. 复用结构，而非样本论文主题。
2. 复用修辞，而非样本论文术语。
3. 复用章节逻辑，而非样本论文章节主题。
4. 复用实验组织，而非样本论文测试系统。
5. 若用户的论文主题不同，围绕用户实际研究对象重写每个占位符，而非围绕样例论文对象。
6. 若用户主题未知，保持占位符通用，不要臆造领域。

  用于用户询问如何编译、清理、重建或排查 ThuThesis LaTeX 项目故障时。

  作为返回起草文本前的最后一道检查，或审阅现有论文章节时的主要评分标准。

  用于第 1 章或独立引言节。

  用于模型、方法或规划章。

  用于章末小结或全文结论。

## 硬规则

1. 保持贯穿本章的一条主问题线。
2. 不要将论文文本写成互不关联的文章列表。
3. 优先采用主题式文献综述，而非按作者罗列。
4. 有意识地使用过渡语：`针对`、`首先`、`其次`、`基于此`、`最后`。
5. 每个技术章节必须按顺序包含 `概述 + 主要内容 + 算例分析 + 本章小结`。
6. `主要内容` 可拆分为多个连续小节，但这些小节共同服务于一个区块：本章的主要方法/模型内容。
7. 四个区块中若缺少任何一个，不要将该章视为结构完整。
8. 每个技术章节的 `概述` 与 `本章小结` 都必须至少约占编译后 PDF 的 2/3 页；这是硬性长度约束，不是风格偏好。
9. 若草稿尚未编译，通过将 `概述` 与 `本章小结` 各自写成多个包含足够问题铺垫、方法链条总结和验证解读的完整段落来近似达到该篇幅。
10. 对硕士论文，第 2-4 章编译后应各约 25 页，除非不可避免，否则不得超过 30 页；这是较强的篇幅预算约束。
11. 对硕士论文，第 2-4 章每章的 `算例分析 + 本章小结` 应合计编译为 10-15 页。
12. 任何编号为 `x.y` 的 `算例分析` 节，其直接下属小节应合理组织在 `x.y.1` 至 `x.y.5` 之内。若内容更多，合并相关分析或将细节移入段落 / 现有五个小节下的中文圆括号子项中。
13. 每次 LaTeX 编译后，检查并报告第 2-4 章及其 `算例分析 + 本章小结` 的页数预算；若无法达到范围，说明原因，而非默默忽略。
14. 不要把完整结论放进图标题或表标题中。
15. 不要不做解释就丢下公式。
16. 实验节不要仅以原始观察结尾；要说明对比证明了什么。
17. 图默认以 PDF 格式插入。
18. 若拟插入或已有的图不是 PDF 格式，应明确提醒用户，而非默默接受。
19. 论文层面的 `结论` 章不得设节或小节标题，无论它是第 5 章还是第 6 章。
20. 论文层面的 `结论` 章应按 开篇总述段 -> 编号贡献点 -> 收尾段 组织。
21. 论文层面结论的每条编号贡献须使用加粗编号 lead sentence 作为独立 LaTeX 段落，然后在下一段继续解释。
22. LaTeX 中 lead sentence 写作 `\textbf{（n）提出了……}`。
23. 每条论文层面结论贡献须遵循 `挑战 -> 方法 -> 结果/验证 -> 价值`，且不得粘贴各章小结。
24. 不要以 `第1章...第2章...` 的方式写摘要。
25. 审阅时，以论文结构和局部过渡为标准评判章节，而非以泛化的散文偏好。
26. 除非用户明确要求该确切主题，否则不要从来源论文引入领域特定名词。
27. 对优化-规划小节，强制保持层次 `x.x -> x.x.1/x.x.2/x.x.3 -> （1）（2）（3） -> 公式组 -> 解释`。
28. `目标函数` 下使用中文圆括号子项如 `（1）投资成本`、`（2）运行成本` 拆分目标块。
29. 每个主要约束小节内部，使用中文圆括号子项如 `（1）火电机组……约束`、`（2）可再生能源……约束` 拆分物理模块。
30. 优化/规划写作下的每个中文圆括号子项必须写成加粗独立 lead，如 `\textbf{（1）投资成本}` 或 `\textbf{（2）电力平衡约束}`。
31. 每个加粗中文圆括号 lead 之后，解释从下一段开始，不要接在同一行。
32. 每个子项后的解释段落必须保持首行缩进，同一子项下的所有后续解释段落也必须保持首行缩进，不要顶格。
33. 对优化-规划目标，不仅要解释公式，还要解释每个块的物理含义以及关键变量、上标、下标的含义（需要时）。
34. 对优化-规划约束，先在 `x.x.2/x.x.3` 层按大系统类别分组，再在 `（1）（2）（3）` 层按物理模块分组。
35. 若用户的论文包含优化/规划模型写作，默认采用本模式，除非用户明确要求不同结构。
36. 写作 LaTeX 显示公式时，`\begin{equation}` / `\begin{align}` 前不要插入额外空行；公式环境应紧跟引入句。
37. `\end{equation}` / `\end{align}` 后始终留一空行；公式块后的叙述必须作为新段落开始，不得在下一行立即开始。
38. 每条编号 LaTeX 显示公式必须有 `\label{eq:...}`。
39. `align` 中每条独立编号的公式行必须有各自的 `\label{eq:...}`；若某行只是延续且不应被引用，则用 `\notag`/\nonumber` 抑制其编号。
40. 引入、解释或后续提及公式时，显式引用为 `式（\ref{eq:...}）`；不要仅使用"上式""如下式"等措辞。
41. 写作 LaTeX 引用时，根据局部句子模式显式区分 `inline` 和 `super` 引用风格。
42. 若引用前有 `文献` 或 `参考文献` 等显式措辞，先切换为 inline 引用风格再引用：`\thusetup{ cite-style = inline }`。
43. 若引用作为某论断或分句的上标参考文献，而非跟在 `文献` 等显式措辞之后，先切换为上标引用风格：`\thusetup{ cite-style = super }`。
44. 每当某处所需引用风格与当前局部设置不同时，显式调用 `\thusetup{ cite-style = ... }`；不要假设正确风格已经激活。

## 输出标准

好的输出应当读起来像规范的工科论文章节：

- 清晰的问题铺垫
- 稳定的节层次
- 密集但可控的技术散文
- 显式的公式与符号处理
- 有节制的图/表引用
- 结果解读回扣论文目标
