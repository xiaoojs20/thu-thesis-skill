# 公式、图表与引用规则

本文件同时覆盖三件事：

1. 公式写作的固定叙述顺序
2. 图、表和正文解读的分工
3. ThuThesis/LaTeX 中常用的落地格式

## 公式写作顺序（公式写作八股）

每条公式的呈现必须遵循以下强制顺序：

1. **引入句**：说明为什么需要该公式，以冒号（`：`）结尾
2. **公式**：带 `\label` 的 `equation` 或 `align` 环境
3. **叙述（显式命名）**：新段落或句子以 `式（\ref{label}）为……` 开头
4. **变量说明**：短式用 `其中`，长式用 `式中：`
5. **用途说明**：说明该公式的建模作用、物理意义或后续用途

不要空降公式，也不要在给出公式后只解释符号而不解释它在论文主线中的作用。

典型顺序可概括为：

1. 先说明"为什么需要这个公式"
2. 给出单个公式或一组公式
3. 若是一组公式，逐条说明各式作用
4. 用 `式中：` 解释变量、函数、集合或上下标
5. 再补一句说明该公式实现了什么建模功能

### 标签与显式引用规则

这是强制性要求，不是建议：

1. 每条编号显示公式必须有 `\label{eq:...}`
2. 单个 `equation` 环境至少包含一个 `\label{eq:...}`
3. 在 `align` 中，每条独立编号的公式行必须有各自的 `\label{eq:...}`
4. 引入、解释或后续提及公式时，必须显式引用为 `式（\ref{eq:...}）`
5. 对于公式组，用 `式（\ref{eq:a}）--式（\ref{eq:c}）` 或 `式（\ref{eq:a}）至式（\ref{eq:c}）` 引入，再解释每条公式的作用
6. 若 `align` 块仅为一条公式的多行推导，则用 `\notag` 或 `\nonumber` 抑制非独立行的编号，仅对需要引用的公式行加标签

正确的 `align` 模式：

```tex
\begin{align}
    & P = \frac{U_s U_r}{X} \sin(\delta_s - \delta_r)
    \label{eq:P}
    \\
    & P_{\max} = \frac{U^2}{X}
    \label{eq:Pmax}
    \\
    & \Delta U = \frac{QX}{U}
    \label{eq:deltaU}
\end{align}

式（\ref{eq:P}）与式（\ref{eq:Pmax}）分别给出了……，式（\ref{eq:deltaU}）给出了……。
```

### 公式组解释规则

对于连续编号公式，不是放出来就结束，而是必须逐条交代。推荐顺序：

1. `式（n）描述状态转移、目标或平衡关系`
2. `式（n+1）定义中间量、漂移项、成本项或风险项`
3. `式（n+2）计算扩展项、约束项或概率项`
4. `式（n+3）给出最终形式、边界条件或闭环关系`

这类写法尤其适合：

1. 随机过程建模
2. 分层优化模型
3. 多约束块规划模型
4. 风险评估公式链

### 公式模式模板

```text
……构建了[模型名称]：
\begin{equation}
  [公式内容]
\label{eq:name}
\end{equation}

式（\ref{eq:name}）为[具体物理含义/约束类型]。
式中：……。
该式实现了……。
```

## `式中：` 规则

1. 以 `式中：` 开头
2. 按变量出现顺序排列
3. 用 `；` 分隔各项
4. 先解释物理含义，再解释角色、范围、上下界或索引含义

模板：

```text
式中：X 表示……；Y 表示……；Z 为……，其上限为……。
```

若公式较复杂，变量说明建议分三层：

1. 全文符号表：高频缩略语、常用变量
2. 章节符号表：本章反复使用的集合、上标、下标、状态量和参数
3. 当前公式后的 `式中：`：只解释新增或易混淆符号

当本章变量很多时，可单列 `本章主要符号表`；不要把大量跨节重复变量全都压到单条公式后解释。

## 公式后还需说明什么

不要停在符号定义，还要说明建模功能。常见落点包括：

1. 描述不确定性
2. 保持时序相关性
3. 量化季节或阶段波动
4. 定义平衡约束
5. 降低计算复杂度
6. 支撑后续场景分析、风险评估或方案比较

简化判断标准是：公式后至少要回答"这个公式在论文主线里具体起什么作用"。

## 图规范

### 功能分工

图主要用于：

1. 背景认知
2. 研究框架
3. 拓扑结构
4. 流程示意
5. 动态过程
6. 平衡曲线
7. 灵敏度分析

可进一步理解为：

1. 引言图回答"为什么研究"
2. 方法图回答"怎么建模或怎么实现"
3. 结果图回答"方法效果如何"

### 标题风格

标题保持名词性，不用结论性表述。良好示例：

1. `某系统拓扑结构`
2. `某方法框架`
3. `某结果对比分析`
4. `某月平衡结果`
5. `某灵敏度分析`

不要把最终结论放进标题，结论应写在正文解释里。

### 正文中的图解读

采用以下模式：

```text
图 x 展示了……。
由图可知，……。
相较于方案 A，方案 B ……。
```

图前通常先说明"展示什么"，图后再说明"看出了什么"和"差异来自什么机制"。

## 表规范

表主要用于：

1. 方案描述
2. 参数设置
3. 结构化结果对比
4. 文献汇总

标题同样保持名词性：

1. `对比方案描述`
2. `关键参数设置`
3. `不同方案的优化结果对比分析`

## 图与表的分工

1. 表承载结构化的数值对比、参数和方案定义
2. 图承载趋势、拓扑、过程、动态行为和机理

若紧凑的表能更好表达结果，就不要强行画图；若需要解释过程变化和机制差异，则优先用图。

## 引用风格

工科论文正文优先采用数字引用。引用既要符合学校模板，也要服从局部句子结构：

1. 若引用前有 `文献`、`参考文献` 等显式措辞，采用 inline 风格
2. 若引用只是为技术论断、事实或结论提供上标支撑，采用 super 风格
3. 当局部所需风格和当前设置不一致时，显式切换 `\thusetup{ cite-style = ... }`

引用应放在所支撑的技术论断之后，而不是孤立悬挂在段末。

## LaTeX 约定

若输出 LaTeX 兼容文本，默认采用：

1. 单行公式 -> `equation`
2. 多行公式 -> `align`
3. 正文中的公式引用 -> `式（\ref{eq:...}）`
4. 每条编号公式必须有 `\label{eq:...}`
5. `align` 中每条独立编号行必须有各自的 `\label{eq:...}`
6. 图 -> `figure`
7. 表 -> `table`
8. 标签前缀：
   - `eq:...`
   - `fig:...`
   - `tab:...`

推荐图环境：

```tex
\begin{figure}
  \centering
  \includegraphics[width=0.5\linewidth]{example-image-a.pdf}
  \caption{示例图片标题}
  \label{fig:example}
\end{figure}
```

推荐做法：

1. 先 `\centering`
2. 再 `\includegraphics`
3. `\caption` 在前，`\label` 在后
4. 图优先使用 PDF 格式
5. 若需并列对比，优先使用 `subcaption`
